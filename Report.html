Bonus: UMA 2.0 protocol

5. Learn about UMA, its main concepts and flows

First, get familiar with the concepts of UMA grant flow.

What is protection API token?

It’s an access token with the OAuth scope uma_protection.

What is permission ticket?
It’s a different type of token for UMA. It is used to associate a request for a particular resource with the corresponding scopes.

How does resource server create permission ticket?

The resource server makes the POST request to the keycloak. The body of the request contains a JSON object to request a permission for single resource identifier or an array of one or more objects.

How does client use permission ticket?

When the client receives the ticket, it needs to get the RPT (Requesting Party Token). To do so, it should send the ticket back to the authorization server.

What should authorization server do with permission ticket?

When the authorization server has received a request for an RPT from a client, it will check whether the client is authorized to get the RPT. Check which permissions the client has, determine a set of requested scopes for resources, evaluate the authorization status -> decide to issue RPT or error.

What is RPT?

It is simply a token with necessary authorization data (permissions).
